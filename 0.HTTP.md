# 0. HTTP

## 목차

## 1. 웹 동작 방식

- 통신하려면 주소와 포트번호를 알아야 함
- TCP/IP
  - 패킷단위
  - IP(internet protocol)
    - IP주소:호스트에 연결된 노드(PC)에 붙여진 번호(주소)
    - IP주소를 바탕으로 데이터를 전송할 때 사용하는 프로토콜
    - 정확한 전달을 보장하지 않아 신뢰성 낮음(중복,분실 등을 체크하지 않음)
    - ipv4(4byte), ipv6(8byte)
  - TCP(transmission control protocol)
    - 3 way handshake 방식을 이용하여 신뢰성을 보장하는 방식
    - 패킷 순서 보장
    - port번호: 해당 메시지를 수신할 어플리케이션을 지정하는 정수
    - 1:1
  - UDP
    - 불특정다수에게 전송
    - ex) 뉴스레터, brodcast
  - 라우터 사이에서는 ip로 작동, 처음/최종 단계에선 tcp동작(end-to-end)
  - http는 tcp/ip 위 단계에서 작동

## 2. HTTP프로토콜

- 통신규약. 통신절차 정의
- 특징
  - 문자기반
  - html 문서 단위
  - 이전의 상태를 기억하거나 유지하지 않음: statelees
    서버에 연결 할 수 있는 자원이 한정적이기 때문
- 절차
  - 연결(connect)
  - 요청(request)
  - 응답(response)
  - 해제(colse)
- HTTP Request
  - Request Line: 엔터(1번줄)로 line인식
    - HTTP메서드, URL, 프로토콜 3가지 정보로 구성(공백으로 구분)
    - HTTP method: GET, POST, PUT, DELETE
  - Request Header
    - 헤더 안에서는 한줄씩 구성, \n문자(빈줄)로 헤더와 body구분
    - 각 줄 안에선 콜론(헤더명:값)
  - Request Body
    - Get은 Body 없음
- HTTP Response
  - Status Line(1번줄):
    - 프로토콜, 상태코드, 설명 -(공백으로 구분)
    - 상태코드
      - 200(OK)/404(Not Found)/500(Internal Server Error)
      - 403(forbbiden권한)
  - Response Header
    - 정보명:정보
    - Content-Type: mime-type (text/img...)
  - Response Body
    - Text로 구성
    - 내용을 서버가 조작(가공) 가능 **=> 백엔드 프로그래밍**
- URL
  - 프로토콜/host/port/resource path/query
  - query string: ?a=b&c=d

## \<form> 태그

- method: http 메서드
  - get(default): url로 값 전달(URL Encoding), 길이 제한
  - post: 비공개(request body)로 값을 전달, 제한x
- action: 전송할 url
- input
  - name,value: query string에서 쓰임

> \* 개발자 도구 network탭 > 요청한 request와 각 헤더 정보, payload에서 전송된 쿼리스트링을 확인할 수 있음
