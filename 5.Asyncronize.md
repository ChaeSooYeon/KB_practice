# 비동기 처리

## 비동기 처리 방법(1) - 콜백함수

비동기코드를 원하는 순서로 실행할 수 있게 하는 방법.  
함수를 인자로 받아오는 것이 특징.

```js
//EX) 회원가입 시나리오
//1. 회원가입 API 호출되면 DB에 회원 정보 저장
//2. 이메일을 보내고
//3. 성공메시지 출력
// 콜백방식으로 작성

const db = [];

//회원가입 API 함수
function register(user) {
  return saveDB(user, (user) => {
    return sendEmail(user, (user) => {
      return getResult(user);
    });
  });
}
// 가입요청한 회원의 정보를 저장하는 API함수
function saveDB(user, callback) {
  db.push(user);
  console.log(`save ${user.name} to DB`);
  return callback(user);
}
// 회원의 정보를 저장 후 회원에게 이메일 전송
function sendEmail(user, callback) {
  console.log(`send to ${user.email}`);
  return callback(user);
}
// 회원에게 가입 성공 메시지 보여주기
function getResult(user) {
  return `success register ${user.name}`;
}

const user = { name: 'kim', email: 'abc@email.com' };
const result = register(user);

console.log(result);
```

=> 콜백지옥이라는 한계

## 비동기 처리 방법(2) - Promise객체

Promise: 처리 상태 정보를 갖고있는 객체를 반환하는 방식

- 미래에 (코드)실행할 것을 약속 한다는 개념
- 이행, 거절(reject), 대기 3가지의 상태를 갖고, 반환한다

```js
function registerByPromise(user) {
  const result = saveDB(user).then(sendEmail).then(getResult);

  console.log(result); //아직 완료 되지 않았으므로 지연(pending) 상태
  return result;
}

const result1 = registerByPromise(user);
//결과가 promise 객체이므로 then()메서드에 함수를 넣어서 결괏값 출력
result.then(console.log);
```

```js
//promiseAll : 넘긴 배열의 순서대로 한번에 실행
const result2 = Promise.all([saveDB(user), sendEmail(user), getResult(user)])
  .catch((err) => new Error(err))
  .finally(() => console.log('완료'));

result2.then(console.log);
```

## 비동기 처리 방법(3) - async/await

async/await 예약어를 사용하는 방법  
**비동기 처리를 동기형식으로 표현**할 수 있는 기법.  
코드 복잡도를 줄이기에 가장 적합한 방식

```js
async function myName(params) {
  return 'suyeon';
} //Promise 객체를 리턴
const res = await myName();
// 'await': 실행상태(성공/대기/실패)를 가지는 Promise 객체의 실행의 완료를 기다림(성공/실패).
//  async 함수 안에서만 사용 가능

console.log(myName()); //Promise{'suyeon'}
console.log(res); // suyeon
```

- await는 반드시 async 함수 내부에서만 동작!
- await를 안쓰면 동기처리 되고, promise를 리턴
