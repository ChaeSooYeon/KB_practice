## 비동기 처리

동기 처리 방식:요청을 순서대로 처리하지 않음  
비동기 처리 방식: 요청을 요청한 순서대로 처리

```javascript
function func1() {
  console.log('A');
}
// function func2() {
//   console.log('B');
// }
function func2() {
  setTimeout(() => {
    console.log('B');
  }, 2000);
}
function func3() {
  console.log('C');
}

func1();
func2();
func3();
```

출력

```
A
C
B
```

비동기 처리 방법

- callback 함수 / promise / **(async/await)**

#### callback 함수

```javascript
function func1() {
  console.log('A');
}
function func2(callback) {
  console.log('B');
  callback();
}
function callback() {
  console.log('C');
}

func1();
func2(callback);
```

출력

```
A
B
C
```

콜백 지옥이라는 한계 있음

#### promise

특징

- 콜백의 depth는 낮아지지만 복잡도는 비슷함
- 성공/실패별로 실행할 콜백 함수를 변수로 줄 수 있음
  - then(result), catch(err)

```javascript
let isTrue = true;
const func1 = new Promise((resolve, reject) => {
  if (isTrue) {
    resolve();
  } else {
    reject();
  }
});

func1
  .then(() => {
    //성공시 동작 함수
    console.log('success');
  })
  .catch((err) => {
    //에러시 동작 코드
    console.log(err);
  });
```

#### **async/await**

async/await 예약어를 사용하는 방법  
**비동기 처리를 동기 스타일로 표현**할 수 있는 기법.  
코드 복잡도를 줄이기에 가장 적합한 방식

Before:

```js
fetch('https://jsonplaceholder.typicode.com/users')
  .then((response) => response.json)
  .then((data) => console.log(data))
  .catch((err) => console.log(err));
```

After:

```javascript
async function init() {
  //async:  자동으로 promise를 생성, 책임은 자바스크립트에게
  // 함수가 promise내부에 생성되고 promise를 리턴함.
  try {
    const res = await fetch('url'); //순서대로
    const data = await res.json(); //실행됨
    console.log(data);
  } catch (err) {
    console.log(err);
  }
}

init();
```

- await는 반드시 async 함수 내부에서만 동작!
- await를 안쓰면 동기처리 되고, promise를 리턴

## DOM조작

DOM이란?
브라우저가 만드는 웹 문서 객체 모델, 구조화된 객체들의 집합

DOM을 이해하는 두 가지 관점

- 구조: HTML은 텍스트지만, 브라우저는 이를 구조화하여 DOM이라는 객체 트리를 만드는 것
- API: DOM API. 자바스크립트를 통해 DOM 객체를 조작하기 위해 브라우저가 제공하는 공식 도구
  - getElementById, querySelectior, appendChild, setAttribute,

==> DOM은 HTML 문서를 표현하는 구조화된 트리이며,
이 트리는 객체들로 구성되어 있고, 자바스크립트로 이 객체에 접근하고 조작할 수 있다.

+) 돔 조작은 HTML 파일을 수정하는 것이 아닌, 브라우저 메모리 상의 DOM 객체를 실시간으로 조작하는 것

## Closure

### 함수를 리턴하는 함수

- 함수를 리턴하는 함수의 사용은 클로저 때문
- ??

```js
function outer() {
  return function () {
    console.log('Hello Function...!');
  };
}
// 호출 1
outer()();
// 호출 2
var fn = outer();
fn();
```
